## Scaling Icon Issue

### Problem
Waybar scaling icon wasn't working. Script exists but failing silently.

### Root Cause
Hyprland v0.55.4 switched to Lua config. Script was using deprecated `hyprctl keyword` syntax.

### Fix Applied
Updated `/home/mattie/.config/hypr/scripts/toggle_scale.sh` to use Lua syntax:
```bash
hyprctl eval "hl.monitor({ output = \"$CURRENT_MONITOR\", scale = $NEW_SCALE })"
```

Instead of old format:
```bash
hyprctl keyword monitor "$CURRENT_MONITOR,preferred,auto,$NEW_SCALE"
```

Now cycles through 75%, 100%, 150% correctly.
